Keyboard Shortcuts

Every shortcut to navigate The Associate Studio like a power user.

The Associate Studio is designed to be driven from the keyboard. Most actions have a shortcut, and the command palette fills the gaps for everything else. This page documents every binding, organized by what part of the interface it controls.

Global

These shortcuts work everywhere in the app, including when the terminal has focus.

ShortcutActionNotes
Ctrl + P Open Command Palette Always intercepted, even in terminal
Ctrl + Shift + Space Toggle Neural Field Mission control overlay with active sessions and system status
Escape Close overlays Closes Neural Field first, then Command Palette (priority order)

Panels

Toggle the three resizable panels that make up the IDE layout.

ShortcutActionNotes
Ctrl + B Toggle Sidebar Left panel (sessions, git, PRs)
Ctrl + Shift + B Toggle Right Panel Context viewer, teams, plans
Ctrl + J Toggle Bottom Panel Git log, output, PRs, issues

Sidebar Views

Switch between sidebar tabs. If the sidebar is closed, these shortcuts also open it.

ShortcutAction
Ctrl + 1 Show Sessions
Ctrl + 2 Show Git
Ctrl + 3 Show PRs

Sessions and Tabs

Manage Claude sessions and navigate between open tabs.

ShortcutActionNotes
Ctrl + N New Session Opens a new Claude CLI terminal tab in the active project
Ctrl + R Resume Session Resumes the current session-view tab as a live terminal
Ctrl + W Close Tab Closes the currently active tab
Ctrl + Tab Next Tab Cycles forward through open tabs
Ctrl + Shift + Tab Previous Tab Cycles backward through open tabs

Projects

Cycle between open projects. Each project maintains its own set of tabs and session state.

ShortcutAction
Ctrl + Shift + → Next Project
Ctrl + Shift + ← Previous Project

Settings and Font

Quick access to settings and font size adjustments. Font size changes apply to the terminal and the entire UI simultaneously.

ShortcutActionNotes
Ctrl + , Open Settings Opens the Settings tab
Ctrl + = Increase Font Size +1px, max 24px
Ctrl + - Decrease Font Size -1px, min 8px
Ctrl + 0 Reset Font Size Resets to 13px default

Bottom Panel

Shortcuts that target specific bottom panel tabs.

ShortcutActionNotes
Ctrl + Shift + G Show Git Log Opens bottom panel and switches to the Git tab

Development

Shortcuts available only in development builds.

ShortcutActionNotes
Ctrl + Shift + D Toggle Debug Panel Dev builds only. Shows internal event log (max 500 entries)

Command Palette

The command palette (Ctrl + P) provides access to over 25 commands, including many that don't have dedicated keyboard shortcuts. Commands are organized into four categories:

CategoryExample Commands
View Toggle Sidebar, Toggle Right Panel, Toggle Bottom Panel, Show Sessions, Show Git, Show PRs, Show Issues, Right Panel: Context, Right Panel: Teams, Right Panel: Plans, Bottom Panel: Git, Bottom Panel: Output, Toggle Neural Field
Session New Claude Session, Resume Session, Close Current Tab, Next Tab, Previous Tab
Project Next Project, Previous Project
Settings Open Settings, Increase Font Size, Decrease Font Size, Reset Font Size

Type to filter commands by name. Each command shows its keybinding (if one exists) on the right side. Press Enter to execute the highlighted command, or Escape to dismiss.

Tip: The command palette is especially useful for actions that only appear there, like switching to specific right panel tabs (Context, Teams, Plans) or specific bottom panel tabs (Output).

Terminal Focus Behavior

When a Claude session terminal (xterm.js) has focus, most keyboard input goes directly to the running process. This is intentional — you need to type freely when working with Claude. But it means shortcuts behave differently depending on where focus is.

What always works in the terminal

A small set of "terminal-safe" shortcuts are intercepted even when the terminal is focused:

  • Ctrl + P — Command Palette (always intercepted)
  • Ctrl + Shift + Space — Neural Field (always intercepted)
  • Ctrl + Tab / Ctrl + Shift + Tab — Tab switching
  • Ctrl + Shift + → / ← — Project cycling
  • Ctrl + Shift + B — Right panel toggle
  • Ctrl + = / - / 0 — Font size

What goes to the terminal

Everything else passes through to the terminal process. This includes:

  • Ctrl + C — Sends SIGINT (interrupt) to the running process, not "copy"
  • Ctrl + V — Pastes into the terminal
  • Ctrl + B — Passes through (does not toggle sidebar from terminal)
  • All regular typing, arrow keys, etc.

Copy from terminal: Use Ctrl + Shift + C to copy selected text from the terminal. The standard Ctrl + C sends an interrupt signal to the process instead.

Input Guard

When focus is in a regular input field or textarea (not the terminal), most shortcuts are suppressed to avoid interfering with text editing. The exceptions are the three "always active" shortcuts:

  • Ctrl + P — Command Palette
  • Ctrl + Shift + Space — Neural Field
  • Escape — Close overlays

This means you can safely type Ctrl + B in a text field without accidentally toggling the sidebar. If you need a shortcut while editing text, open the command palette and run the action from there.
