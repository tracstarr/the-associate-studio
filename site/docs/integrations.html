Integrations

Connect GitHub, Linear, and Jira for a unified development workflow.

The Associate Studio pulls data from your existing tools and surfaces it alongside your Claude sessions. All integrations are optional — the IDE works fine without any of them — but connecting even one transforms the sidebar and bottom panel into a real-time project dashboard.

GitHub Integration

GitHub is the most full-featured integration. Once connected, the sidebar's PR view shows open pull requests, the Issues tab lists repository issues, and the gh CLI becomes available inside Claude sessions for branch creation, PR submission, and review workflows.

There are two ways to authenticate. The OAuth Device Flow is recommended because it grants scoped access and can be revoked from your GitHub settings at any time. A Personal Access Token works just as well if you prefer simplicity or are behind a corporate proxy that blocks the device flow.

Method 1: OAuth Device Flow Recommended

Prerequisites

You need a GitHub OAuth App with the Device Flow enabled. This is a one-time setup — the same app can be shared across your team.

1

Create a GitHub OAuth App

Go to github.com/settings/developers and click New OAuth App. Fill in any name and homepage URL. The callback URL doesn't matter for the device flow — use http://localhost.

2

Enable Device Flow

On the app's settings page, check the "Enable Device Flow" checkbox and save. Without this, the device code request will fail with a 404.

3

Copy the Client ID

Copy the Client ID (not the Client Secret — that isn't needed for device flow). Paste it into The Associate Studio's Settings panel under GitHub → Client ID.

4

Start the flow

Click "Start Device Flow" in the Settings panel. The app requests a device code from GitHub with repo and read:org scopes. A user code appears on screen (e.g., ABCD-1234), and your browser opens to github.com/login/device.

5

Enter the code on GitHub

Type the user code into the GitHub verification page and authorize the app. Back in The Associate Studio, polling completes automatically — the token is exchanged, stored in Windows Credential Manager, and the gh CLI is configured via gh auth login --with-token.

Method 2: Personal Access Token Alternative

Quick setup for users who prefer a classic PAT

  1. Go to github.com/settings/tokens and create a new token (classic or fine-grained).
  2. Select at minimum the repo and read:org scopes. Without repo, PR and issue data won't load.
  3. Copy the token and paste it into Settings → GitHub → Personal Access Token.
  4. Click Save. The token is stored in Windows Credential Manager and piped to gh auth login --with-token.

What gets stored: The access token is saved under the service name the-associate-studio with the key github-token in Windows Credential Manager. It is never written to settings.json or any file on disk.

Linear Integration

Linear integration brings your team's issues into the sidebar so you can reference them while working with Claude. Authentication uses a personal API key — no OAuth redirect dance required.

Setup

  1. Go to linear.app/settings/api and create a new personal API key.
  2. Paste the key into Settings → Linear → API Key.
  3. Click Verify. The app makes a GraphQL request to https://api.linear.app/graphql with the query { viewer { name } }. If the response includes your name, the key is valid and gets stored in Windows Credential Manager.

What it enables

Once connected, the Issues sidebar view lists your Linear issues with filtering by:

  • Status — Backlog, Todo, In Progress, Done, Cancelled
  • Team — filter by Linear team
  • Assignee — show only your issues or all team issues

The API returns up to 50 issues per request. Issues display their identifier (e.g., ENG-142), title, status, and assignee.

Note: Linear's API key acts as a bearer token. It's sent in the Authorization header directly (not as Bearer <token> — Linear accepts both formats). The key is stored under linear-api-key in Windows Credential Manager.

Jira Integration

Jira integration supports Jira Cloud instances. It uses API token authentication with Basic Auth, which is the simplest reliable method for a desktop app.

Setup

  1. Go to Atlassian API Tokens and create a new token.
  2. In Settings → Jira, enter three fields:
    • Base URL — your Jira Cloud URL, e.g. https://yourcompany.atlassian.net
    • Email — the email address associated with your Atlassian account
    • API Token — the token you just created
  3. Click Verify. The app sends a GET request to {base_url}/rest/api/3/myself using Basic Auth (base64-encoded email:token). If the response includes your display name, the connection is confirmed.

Why not OAuth?

Jira's OAuth 2.0 (3LO) flow requires a redirect URI and a local HTTP server to catch the callback. For a native desktop app that doesn't run a web server, this adds significant complexity. Basic Auth with API tokens is the officially documented approach for server and desktop applications, and it works without any additional infrastructure.

Storage split: The API token goes into Windows Credential Manager (key: jira-api-token). The base URL and email are non-sensitive metadata and are stored in settings.json.

Claude CLI Hooks

Unlike the integrations above, Claude CLI hooks are configured automatically when The Associate Studio launches. There is nothing to set up manually. The hooks allow the IDE to track Claude session lifecycle events in real time.

How it works

On first launch, the app writes a small Node.js script to ~/.claude/theassociate/hook.js and registers it in Claude's ~/.claude/settings.json under the hooks key. When Claude CLI fires a hook event, it pipes the event JSON to stdin of the hook command. The script reads stdin and appends the JSON line to ~/.claude/theassociate/hook-events.jsonl.

Tracked events

Five hook events are registered:

SessionStart SessionEnd SubagentStart SubagentStop Stop

These events let the IDE show:

  • Live session indicators — a green dot next to sessions that are currently running
  • Subagent tracking — when Claude spawns subagents (e.g., for tool use), the UI reflects it
  • Session duration — start/end timestamps enable elapsed time display

The hook script

The hook.js file is intentionally minimal — it reads stdin to completion, then appends the line to the JSONL file:

'use strict';
var d = '';
process.stdin.on('data', function(c) { d += c; });
process.stdin.on('end', function() {
  var line = d.trim();
  if (!line) return;
  var path = require('path');
  var fs = require('fs');
  var home = process.env.USERPROFILE || process.env.HOME || '';
  var out = path.join(home, '.claude', 'theassociate', 'hook-events.jsonl');
  fs.appendFileSync(out, line + '\n');
});

File watcher

The Rust backend watches ~/.claude/theassociate/hook-events.jsonl for changes. When new lines appear, it parses them and emits Tauri events that the React frontend consumes to update session state. The watcher tracks its file offset so it only reads newly appended lines, not the entire file on every change.

The JSONL file is automatically truncated to the last 1,000 lines when it grows beyond that limit, preventing unbounded disk usage.

Removing hooks

Hooks can be removed cleanly from the Settings panel. The removal process strips the hook entries from ~/.claude/settings.json without affecting any other hooks you may have configured. The ~/.claude/theassociate/ directory is left in place (it contains your hook event history), but you can delete it manually if you want a completely clean slate.

Security Note

Every secret — GitHub tokens, Linear API keys, Jira API tokens — is stored in Windows Credential Manager under the service name the-associate-studio. This is the same credential store used by Git Credential Manager, VS Code, and other desktop applications on Windows.

SecretCredential KeyStorage
GitHub access tokengithub-tokenWindows Credential Manager
Linear API keylinear-api-keyWindows Credential Manager
Jira API tokenjira-api-tokenWindows Credential Manager
Jira base URLN/Asettings.json (not sensitive)
Jira emailN/Asettings.json (not sensitive)

Windows Credential Manager encrypts stored credentials using DPAPI (Data Protection API), which ties encryption to your Windows user account. Credentials are accessible only to processes running under your user profile. They are never written to settings.json, log files, or any other plaintext location.

Important: If you export or share your settings.json file, it will not contain any tokens or API keys. Integration secrets must be re-entered on each machine.
