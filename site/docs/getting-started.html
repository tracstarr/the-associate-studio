Getting Started

Everything you need to install and run The Associate Studio on your Windows machine.

Prerequisites

The Associate Studio is a native Windows desktop application. You'll need the following before running it:

RequirementVersionPurpose
Windows10 or 11 (x64)Operating system
Node.js20.x+Frontend build tooling
Rust1.83+Backend compilation
MSYS2 + MinGWLatestC compiler for native dependencies
Claude Code CLILatestThe AI engine that powers sessions
gh CLILatest (optional)GitHub PR and issue integration

Installation

1

Install the Rust Toolchain

Install rustup and configure the GNU target (no Visual Studio required):

winget install Rustlang.Rustup
rustup toolchain install stable-x86_64-pc-windows-gnu
rustup default stable-x86_64-pc-windows-gnu
2

Install MSYS2 and MinGW GCC

This provides the C compiler needed for native Rust dependencies like libgit2:

winget install MSYS2.MSYS2
# Then open MSYS2 shell and run:
pacman -S mingw-w64-x86_64-gcc
3

Clone and Install Dependencies

git clone <repo-url> the-associate-studio
cd the-associate-studio
npm install
4

Set Your PATH

This must be done in every terminal session before building:

export PATH="/c/msys64/mingw64/bin:$HOME/.cargo/bin:$PATH"
export PKG_CONFIG_PATH="/c/msys64/mingw64/lib/pkgconfig"
5

Run in Development Mode

npm run tauri dev

This starts the Vite dev server with hot module replacement for the frontend and compiles the Rust backend. First build takes 4–6 minutes; subsequent builds are 15–30 seconds.

Tip: For frontend-only development, run npm run dev to get a browser preview at http://localhost:1420 without compiling Rust.

Building for Release

npm run tauri build
# Outputs: src-tauri/target/release/bundle/

This produces a standalone .exe plus an NSIS installer. The release binary is approximately 10MB — far smaller than an equivalent Electron application.

First Launch

When The Associate Studio starts for the first time, it automatically:

  • Installs Claude CLI hooks for live session tracking (fully reversible)
  • Creates the ~/.claude/theassociate/ directory for hook event storage
  • Starts watching ~/.claude/ for real-time data updates

No manual configuration is required. Open a project folder and start a new Claude session to begin.
